# セグロット在庫連携アプリ（最小形）

セグロット倉庫がスマホで入出庫を入力し、フェリクロスがブラウザで在庫の反映を確認できるアプリです。
現状のExcel＋メール往復を置き換える「日次入出庫の最小形」です。

- セグロット（writer）: 入荷／出荷を入力。カートン・個数のどちらでも入力でき、内部では個数に換算して記録。
- フェリクロス（viewer）: 在庫と当日の動きを確認。
- 谷口さん（admin）: 入力・確認の両方。
- 在庫数 = 期首残 ＋ 入荷合計 − 出荷合計（個数で統一）。商品マスタは現行Excel（2605シート）の160商品を取り込み済み。

---

## まずローカルで動作確認

Python が入っている前提です。このフォルダで以下を実行します。

```
pip install -r requirements.txt
python run.py
```

ブラウザで `http://127.0.0.1:8000` を開きます。
動作確認用のログインは次の通りです（本番では必ず変更します。下の「認証の設定」参照）。

- セグロット役: ユーザー名 `segrot` / パスワード `demo`
- フェリクロス役: ユーザー名 `felicross` / パスワード `demo`

`segrot` で入ると入力画面、`felicross` で入ると確認画面になります。

## クラウド（PaaS）へ公開する手順

セグロットが社外からスマホでアクセスするため、インターネット上に公開します。
ここでは Render を例にします（Railway や Fly.io でもほぼ同様）。サーバ構築・OS管理は不要です。

1. このフォルダ一式を GitHub のリポジトリにアップロードする（`.gitignore` により `*.db` と `.env` は除外されます）。
2. Render（https://render.com）にアカウント登録し、「New → Web Service」で上記リポジトリを連携。
3. ビルド設定: Build Command は `pip install -r requirements.txt`、Start Command は `uvicorn app.main:app --host 0.0.0.0 --port $PORT`（同梱の `Procfile` でも認識されます）。
4. 無料プランで開始可能。常時起動が必要になったら有料プラン（月数百円〜千円程度）に変更。
5. デプロイ完了すると `https://〜.onrender.com` のURLが発行されます。これをセグロットに伝えます。

## 認証の設定（重要・本番前に必須）

動作確認用の `demo` パスワードのままでは公開しないでください。
パスワードは平文で保存せず、sha256ハッシュで設定します。

1. パスワードのハッシュを作る（手元のPCで実行。決めたパスワードを各自が入力）:

```
python -c "import hashlib,getpass;print(hashlib.sha256(getpass.getpass('password: ').encode()).hexdigest())"
```

2. PaaS（Renderなど）の「Environment（環境変数）」に、次の2つを設定します。
   パスワードのハッシュ値をここに入れます（平文は入れません）。

   - `APP_LOGIN_USERS`
     ```
     [{"user":"segrot","role":"writer"},{"user":"felicross","role":"viewer"},{"user":"daiichiro","role":"admin"}]
     ```
   - `APP_LOGIN_SECRETS`
     ```
     {"segrot":"<segrotのハッシュ>","felicross":"<felicrossのハッシュ>","daiichiro":"<adminのハッシュ>"}
     ```

3. 環境変数を設定すると、`demo` のフォールバックは使われなくなり、設定した値でログインします。

注意: ユーザー名やロールはこのファイルに書いて構いませんが、パスワード（およびそのハッシュ）はソースコードやGitには絶対に含めないでください。必ずPaaSの環境変数に設定します。

## データについて

- 在庫データは `segrot_inventory.db`（SQLite）に保存されます。
- PaaSによっては再デプロイでファイルが消える場合があるため、本番運用前に「永続ディスク」の設定（Renderなら Disks）を行うか、PostgreSQLへの移行を検討します。これは公開後の次ステップで対応します。

## FBA発送報告（実装済み・Amazon登録は未接続）

セグロット（writer）・谷口さん（admin）は「FBA発送報告」タブから、FBAへ発送する内容を報告できます。

- 発送する商品とカートン数を入力（複数行、入数から個数を自動計算）。
- 箱ごとのサイズ（縦・横・高さ）・重量を入力（Amazon登録に必要な情報）。
- 配送業者（自社便の場合）・備考を入力。
- 発送後、箱ごとに追跡番号を追記できる（発送時点ではまだ分からないため後入力可）。

**現時点ではAmazonへの登録は行われません。** ここで入力したデータは構造化して保存されるのみです。SP-API（Fulfillment Inbound API）の認証情報が整ったら、このデータを使ってAmazonへの実際の登録処理を追加する予定です（次フェーズ）。

## バーコード（JAN）読み取りについて

入力画面に「バーコードで商品を読み取る」ボタンがあります。スマホのカメラで商品のJANコードを読むと、該当商品が自動で選択されます（NextengineマスタのJANを取り込み済み）。

JAN紐付けの状況:
- 全151商品中、147商品にJANを登録済み（主力のピントキッズ各色を含む）。
- 残り4商品はJANが未発番のため、従来どおりリストから選んで入力します。
- サンリオコラボ等で同一JANが複数行に分かれていた6商品は、同一商品として1つに統合済み（期首残は合算）。

重要な注意:
- カメラ機能は HTTPS 接続でのみ動作します。ローカル（http://127.0.0.1）ではカメラが起動しないため、バーコード読み取りは「クラウド公開後（HTTPSのURL）」で使えます。ローカルでは検索・選択での入力をご利用ください。
- ローカルでも、JANを手で検索欄に入れて選ぶことは可能です。
